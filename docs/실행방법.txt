[부산 지역 맛집 리스트 및 교통편 통합 웹서비스 실행방법 및 개발환경]

1. 프로젝트 개요

본 프로젝트는 부산 지역 음식점 정보를 기반으로 음식점 목록, 상세 정보, 리뷰, 평점, 네이버 지도, 지하철, 버스, 공영주차장 정보를 통합 제공하는 웹 애플리케이션이다.

프론트엔드는 React 기반으로 구현하였고, 백엔드는 Spring Boot 기반 REST API 서버로 구현하였다. 데이터베이스는 MySQL을 사용하였으며, 공공데이터 API와 네이버 지도 API를 활용하여 실제 데이터 기반 서비스를 구성하였다.

본 프로젝트는 단순 음식점 CRUD 서비스가 아니라, 실제 부산 지역 음식점 데이터를 바탕으로 음식점 정보와 방문 편의 정보를 함께 제공하는 통합형 웹서비스를 목표로 한다.

사용자는 메인 화면에서 음식점 목록을 확인하고, 음식점명 검색과 지역 필터를 통해 원하는 음식점을 찾을 수 있다. 상세 페이지에서는 음식점 주소, 전화번호, 운영시간, 메뉴, 지도, 교통편, 리뷰 작성 및 리뷰 목록을 확인할 수 있다.

특히 상세 페이지의 교통편 영역에서는 음식점 좌표를 기준으로 가까운 지하철역, 버스정류장, 공영주차장 정보를 함께 제공하여 실제 방문 가능성을 판단할 수 있도록 구성하였다.

6. 데이터베이스 준비

1) MySQL 실행

MySQL 서버를 먼저 실행한다.

2) 데이터베이스 생성

아래 SQL을 실행하여 데이터베이스를 생성한다.

CREATE DATABASE busan_food_db
DEFAULT CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;

3) application.properties 설정 확인

Spring Boot 프로젝트의 src/main/resources/application.properties 파일에서 아래 설정을 확인한다.

spring.application.name=busan-food-backend
server.port=8080

spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
spring.datasource.url=jdbc:mysql://localhost:3306/busan_food_db?serverTimezone=Asia/Seoul&characterEncoding=UTF-8
spring.datasource.username=root
spring.datasource.password=본인비밀번호

mybatis.mapper-locations=classpath:/mapper/*.xml
mybatis.type-aliases-package=com.project.food.dto
mybatis.configuration.map-underscore-to-camel-case=true

4) 주요 확인 사항

- MySQL 서버가 실행 중인지 확인
- busan_food_db 데이터베이스가 생성되어 있는지 확인
- application.properties의 username/password가 본인 PC 환경과 맞는지 확인
- mapper XML 경로가 실제 mapper 파일 위치와 일치하는지 확인

Spring Boot 서버 실행 후 아래 주소에 접속한다.

http://localhost:8080/api/public-food/import

정상 실행 시 다음 메시지가 출력된다.

50개 음식점 데이터 저장 완료

이 과정은 부산맛집 공공데이터 API에서 음식점 데이터를 가져와 MySQL restaurants 테이블에 저장하는 과정이다.

공공데이터 import 이후에는 다음 API를 통해 저장된 음식점 목록을 확인할 수 있다.

http://localhost:8080/api/restaurants

import 기능은 초기 데이터 적재용으로 사용한다. 이미 데이터가 저장되어 있는 경우 중복 저장 여부를 확인해야 한다.

13. 오류 발생 시 확인사항

[React 화면이 빈 화면일 경우]
1) npm run dev가 정상 실행 중인지 확인
2) package.json이 있는 폴더에서 실행했는지 확인
3) 개발자도구 Console 오류 확인
4) api.js export/import 오류 확인
5) React Router 경로 설정 확인

[localhost:5173 접속이 안 될 경우]
1) VS Code 터미널에서 npm run dev 실행 여부 확인
2) Vite 개발 서버가 정상 실행 중인지 확인
3) 터미널에 출력된 Local 주소 확인
4) package.json 위치 확인

[localhost:8080 연결 거부 시]
1) Spring Boot 서버 실행 여부 확인
2) Eclipse Console 오류 확인
3) server.port=8080 확인
4) MySQL 연결 오류 여부 확인

[API 500 오류 발생 시]
1) Eclipse Console 오류 확인
2) Mapper XML 쿼리 확인
3) DTO getter/setter 확인
4) DB 테이블 및 컬럼명 확인
5) Null 값 처리 여부 확인

[지도 출력이 안 될 경우]
1) 네이버 지도 API Client ID 확인
2) index.html script 확인
3) window.naver 로딩 여부 확인
4) 지도 생성 시점이 useEffect 이후인지 확인

[리뷰 수가 반영되지 않을 경우]
1) /api/restaurants/{id}/stats API 확인
2) reviews 테이블 데이터 확인
3) 평균 평점 계산 쿼리 확인
4) 리뷰 등록 후 loadStats 함수 재호출 여부 확인

[교통편이 부정확하게 나올 경우]
1) 음식점 lat, lng 값 확인
2) 지하철역 좌표 목록 확인
3) 부산BIMS API 응답 확인
4) 공영주차장 API 응답 확인

15. 실제 개발 중 발생한 오류 및 해결 내용

[React 빈 화면 문제]
- 원인: api.js에서 joinMember 함수 export 누락 또는 import 경로 오류
- 해결: api.js export 함수명과 각 페이지 import명을 일치시킴
- 결과: localhost:5173 메인 화면 정상 출력

[npm ENOENT 오류]
- 원인: package.json 파일이 없는 폴더에서 npm run dev 실행
- 해결: package.json이 있는 frontend 프로젝트 폴더로 이동 후 npm run dev 실행
- 결과: Vite 개발 서버 정상 실행

[Spring Boot 500 오류]
- 원인: stats API에서 리뷰 데이터가 없을 때 AVG 값 null 처리 문제 발생
- 해결: SQL 쿼리에서 COUNT, AVG 기본값 처리 및 Service/Mapper 수정
- 결과: /api/restaurants/{id}/stats 정상 응답

[지도 API 로딩 문제]
- 원인: window.naver 객체가 생성되기 전에 지도 생성 코드 실행
- 해결: useEffect 내부에서 window.naver와 restaurant 데이터 존재 여부 확인 후 지도 생성
- 결과: 상세 페이지 지도 정상 표시

[배달 플랫폼 버튼 문제]
- 원인: 배달의민족, 쿠팡이츠 개별 가게 URL 확보가 어렵고 단순 검색 버튼은 실효성이 낮음
- 해결: 배달 플랫폼 버튼 삭제 후 네이버 지도 길찾기 및 교통편 기능 강화
- 결과: 상세 페이지 정보 구조 개선

[지하철역 누락 문제]
- 원인: 덕포역 등 일부 부산 지하철역 좌표가 누락되어 가까운 역 계산이 부정확함
- 해결: 부산 1호선, 2호선, 3호선, 4호선, 동해선, 부산김해경전철 주요 역 좌표 추가
- 결과: 지하철 교통편 계산 정확도 향상

[주차 요금 표시 문제]
- 원인: 월주차 요금이 출력되어 음식점 방문 사용자에게 부적절하게 보임
- 해결: 10분 요금, 추가 요금, 일 최대 요금 중심으로 표시 방식 수정
- 결과: 실제 방문자 관점에 맞는 주차 정보 제공


16. 프로젝트 특징

본 프로젝트는 단순 음식점 CRUD 서비스가 아니라 실제 공공데이터와 지도 API를 활용한 통합형 웹서비스이다.

주요 특징:
- 부산 지역 실제 음식점 데이터 사용
- React와 Spring Boot 분리 구조
- MySQL 데이터베이스 연동
- 리뷰 작성 및 평점 계산
- 네이버 지도 API 연동
- 부산BIMS API 기반 버스정류장 계산
- 부산 공영주차장 API 기반 주차 정보 제공
- 지하철역 좌표 기반 가장 가까운 역 계산
- 상세 페이지에서 지도, 교통편, 메뉴, 리뷰 통합 제공
- 사용자 방문 의사결정에 필요한 정보 통합 제공

본 프로젝트는 게시판 중심의 단순 CRUD 프로젝트보다 실제 서비스에 가까운 구조를 가진다. 음식점 정보뿐 아니라 교통편, 지도, 주차 정보까지 결합하여 실사용성을 강화하였다.

18. 최종 정리

본 프로젝트는 부산 지역 음식점 정보를 기반으로 음식점 목록, 상세 정보, 리뷰, 평점, 네이버 지도, 지하철, 버스, 공영주차장 정보를 하나의 웹서비스에서 통합 제공하는 프로젝트이다.

프론트엔드는 React와 Vite를 활용하여 구성하였고, 백엔드는 Spring Boot와 MyBatis 기반 REST API 서버로 구현하였다. 데이터베이스는 MySQL 8.0.34를 사용하였으며, 실제 부산맛집 공공데이터 API를 통해 음식점 데이터를 저장하였다.

또한 부산BIMS 버스정보 API, 부산 공영주차장 API, 네이버 지도 API를 연동하여 사용자가 음식점을 선택하고 실제 방문하기까지 필요한 정보를 상세 페이지에서 확인할 수 있도록 구성하였다.

본 프로젝트를 통해 React, Spring Boot, MySQL, REST API, 공공데이터 API, 지도 API 연동 과정을 종합적으로 경험하였으며, 향후 AWS 배포와 관리자 기능을 추가하면 포트폴리오로 활용 가능한 수준의 프로젝트로 발전시킬 수 있다.
